项目经历 (Project Experience)

1. 医院血液中心库存决策优化系统 (SRTP科研训练 | 核心成员) 2025.05 – 2026.05

  • 业务背景: 针对三甲医院血液中心“过期浪费”与“缺货短缺”双重风险，探索基于数据驱动的库存订货策略。
  • 模型构建: 设计了SEO(先预测后优化)与JEO(端到端联合优化)两套方案。在JEO方案中，基于DeepSets架构设计神经网络，直接从历史申请数据映射到最优订货量。
  • 策略评估: 引入报童模型(Newsvendor Loss)作为损失函数，通过tau参数化模拟不同缺货/积压成本权重。
  • 结果量化: 实验表明，端到端优化(JEO)在业务成本控制上显著优于传统“先预测后优化”方法，并引入Prescriptive Power指标验证了模型相比SAA方法的决策质量提升。

  • 策略优化: 设计维护核心高波动长影线、小市值及人气动量等策略，通过特征工程持续优化模型。
  • 系统开发: 基于 MySQL 构建自动化行情库；开发市值过滤及 Alpha 评价模块实现实时风险预警。

3. 浙里淘书 (zook) · 校园二手教材交易平台 (小组项目) 2026.04 – 2026.06

  • 项目简介: 面向浙江大学师生的二手教材 C2C 交易平台，涵盖书籍发布/审核、检索、收藏、下单、支付、评价完整交易闭环。
  • 技术架构: Vue 3 + Vite + Composition API / Flask + SQLite + Socket.IO / DeepSeek AI API。
  • 前端开发: 基于 Vue 3 Composition API 构建 18 个页面 + 45+ 组件的前端 SPA，使用路由懒加载与 composables 实现逻辑复用。
  • 后端设计: 后端采用 Flask Blueprint 模块化架构，拆分为 12 个业务蓝图，实现关注点分离；设计 13 张 SQLite 数据表支撑业务数据持久化。
  • 核心功能: 集成 Socket.IO 实现实时消息推送与在线状态管理，支持即时聊天和系统通知。
  • 智能与认证: 对接浙大教务系统抓取课表，自动匹配平台二手书资源；接入 DeepSeek AI 构建智能教材助手；实现浙大 CAS 统一身份认证与 JWT 会话管理。
